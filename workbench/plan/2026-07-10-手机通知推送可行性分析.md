# 灵动岛消息推送到手机 —— 可行性分析

> 日期：2026-07-10 · 主题：把 Eureka 灵动岛的通知（任务完成 / 等待授权 / 健康提醒）推送到手机（iOS / Android）
> 结论：**纯可行性分析**，不含实现。推荐走「HTTP 推送到中转服务」，蓝牙不推荐。

## 一、需求

Eureka 是 macOS 菜单栏应用，会把本地 Claude / Codex / opencode / Grok / Antigravity 的任务事件以灵动岛卡片呈现。希望这些「值得通知」的事件也能推送到手机，尤其是**人离开工位时**：
- 任务完成（成功）
- 任务出错 / 中断
- **等待授权**（工具权限确认，最需要离屏提醒——不回来就卡住）
- 健康提醒（连续 vibe coding 过久 / 深夜工作）

## 二、传输方式可行性对比

| 方式 | 可行性 | 说明 |
|---|---|---|
| **HTTP 推送到中转服务** | ✅ 最优 | Mac 侧只发一个 HTTPS `POST`（`URLSession`，零依赖）；手机装对应 app 收推送。跨 iOS/Android、离开工位也能收、后台可靠。与本项目「零第三方依赖」铁律完全契合。 |
| **蓝牙 BLE** | ❌ 最差 | 两端都要自写 app；iOS 后台 BLE 不可靠、会被系统挂起；范围仅 ~10m —— 一离开工位就断，恰好废在最该提醒的场景；Apple ANCS 是反方向（手机→配件），帮不上忙。 |
| **原生 Apple（无中转）** | ❌ 不可行 | Mac app 无法直接给 iPhone 发通知，必须自建 iOS app + 自有 APNs 服务器。太重。macOS 本地通知也不会同步到 iPhone。 |
| **邮件 / 短信** | ⚠️ 次选 | SMTP 不便；短信要 Twilio 等付费网关。能做但重、延迟高、不像通知。 |

**核心洞察**：可行的方向本质都是「发一个 GET/POST」，所以**一套通用 Webhook 引擎 + 各家预设**即可全覆盖，无需为不同服务分别写代码，也无需任何手机端 app 开发。

## 三、中文社区的推送应用/服务（重点）

| 服务 | 接收端 | 开源/自建 | API 形态 | 费用 | 适合谁 |
|---|---|---|---|---|---|
| **Bark** | iOS | ✅ 开源、国内作者、可自建（bark-server） | `GET api.day.app/<key>/<标题>/<正文>` | 免费 | **iOS 单人首选**，最简单 |
| **Server酱·Turbo（方糖）** | 微信 / 企业微信 / 钉钉群 / 飞书群 / 客户端 | 服务端闭源（有开源生态 WeComChan） | `GET sctapi.ftqq.com/<SENDKEY>.send?title=&desp=`（新版 sendkey 走 `push.ft07.com`） | 免费+付费 | **想用微信直收、不装新 app**（人人有微信） |
| **PushDeer** | iOS/Mac/Android/快应用 | ✅ 开源、Docker 一键自建、不依赖微信 | `GET api2.pushdeer.com/message/push?pushkey=&text=` | 自用免费 | **数据完全自主可控** |
| **WxPusher** | 全平台（安卓/iOS/鸿蒙/桌面/微信） | 客户端开源、服务端托管 | `POST wxpusher.zjiecode.com/api/send/message`（JSON） | 永久免费 | **全平台且免费** |
| **PushPlus（推送加）** | 微信 | 闭源托管 | 一对一 token 调用 | 免费含广告 | 快速上手、多渠道聚合 |
| **企业微信/钉钉/飞书 自定义群机器人** | 对应 IM app | —（IM 官方） | `POST` webhook JSON | 免费 | **团队/公司环境**，已装 IM 即可 |

**国际对照**：ntfy（开源可自建，iOS+Android）、Pushover（付费、精致）、Telegram（国内需代理，不宜做国内默认）。

## 四、对本项目（Eureka）的可行性结论

- 上表每一个都只是一次 HTTP GET/POST → 与「零依赖」铁律完全契合（`Foundation` 的 `URLSession` + `JSONSerialization` 即可，无需任何库）。
- **一套「通用 Webhook + 预设」即可全部覆盖**：预设内置各家的 URL 模板 / 请求头 / 正文字段名，用户只填自己的 key/token。
- 挂载点现成：`AppDelegate.applyToUI` 的 `taskFinished` / `taskWaiting` 分支（已带 `isStale` 过滤和 `settings.notify*` 分类开关）+ `WellnessMonitor` 发射闭包。
- 遵循既有隐私约定：**opt-in、默认关**；token 存**钥匙串**（复用 `EurekaSync/KeychainStore`），不进 UserDefaults；失败静默、只在设置页显示一行提示。
- **不需要写任何手机端 app**。

## 五、面向国内用户的默认推荐

- **iOS 单人** → Bark（最简单，可自建）
- **想用微信收、不装新 app** → Server酱 或 WxPusher（免费）
- **团队/已用 IM** → 企业微信 / 钉钉 / 飞书 群机器人
- **要完全自主可控** → PushDeer 自建（或 ntfy 自建）

## 六、后续（若要实现）

轻量方案：新增 opt-in 的 `PhoneNotifyService`（形如 `SyncService`：私有队列 + 钥匙串读取 + `URLSession` POST），在设置页加「手机通知」区（渠道预设选择 / URL / SecureField token / 分事件开关），在 `applyToUI` 的两个分支 + wellness 闭包处调用。预设优先：Bark / Server酱 / WxPusher / 群机器人 / ntfy / 通用 Webhook。
